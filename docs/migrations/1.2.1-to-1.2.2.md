# ModelSync 1.2.1 to 1.2.2 Migration Guide

## Summary

ModelSync 1.2.2 is a release workflow reliability correction. Consumer source changes are not required.

| Question | Answer |
|---|---|
| Consumer code migration required | No |
| Package update required | Yes |
| Behavioral API change | No |
| Workflow/release reliability correction | Yes |

## Recommended Action

Update all ModelSync packages together:

```bash
dotnet add package UmbrellaFrame.ModelSync.Core --version 1.2.2
dotnet add package UmbrellaFrame.ModelSync.SqlServer --version 1.2.2
dotnet add package UmbrellaFrame.ModelSync.MySql --version 1.2.2
dotnet add package UmbrellaFrame.ModelSync.PostgreSQL --version 1.2.2
dotnet add package UmbrellaFrame.ModelSync.SQLite --version 1.2.2
dotnet add package UmbrellaFrame.ModelSync.Analyzers --version 1.2.2
```

## Unpublished 1.2.1 Tag

Version 1.2.1 was tagged but was not published to NuGet because the required
MySQL/MariaDB integration gate did not pass. Version 1.2.2 supersedes the
unpublished 1.2.1 tag.

## Runtime Compatibility

No ModelSync runtime API was removed or changed for consumers. Existing 1.2.0 and 1.2.1-compatible source remains compatible with 1.2.2.
