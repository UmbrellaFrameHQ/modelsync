# Migrating from ModelSync 1.2.3 to 1.3.0

ModelSync 1.3.0 is source-compatible with 1.2.3 for existing public APIs.

## Required Changes

No required code changes are expected.

## Optional Improvements

Use `RunWithResultAsync()` with the new report renderers:

```csharp
var result = await runner.RunWithResultAsync(cancellationToken);

File.WriteAllText(
    "modelsync-migration-report.md",
    MigrationExecutionMarkdownReport.Create(result, "ModelSync Deployment Report"));

File.WriteAllText(
    "modelsync-migration-report.json",
    MigrationExecutionJsonReport.Create(result));
```

Recommended deployment behavior:

1. Save both reports as CI/deployment artifacts.
2. Fail the deployment when `result.Succeeded` is false.
3. Review failed batch index and category before rerunning.
4. Avoid sharing reports publicly when SQL scripts contain business-sensitive content.

## CLI Introduction

`UmbrellaFrame.ModelSync.Cli` is introduced as a .NET tool over the existing migration runner:

```bash
dotnet tool install --global UmbrellaFrame.ModelSync.Cli --version 1.3.0
modelsync validate --scripts ./Database/Scripts
modelsync run --provider sqlite --connection "Data Source=modelsync-preview.db" --scripts ./Database/Scripts --dry-run
```

`modelsync validate` does not connect to a database. `modelsync run --dry-run` connects and compares registered scripts, but does not apply SQL.

## Oracle Preview Note

`UmbrellaFrame.ModelSync.Oracle` is publicly available on NuGet in 1.3.0, but remains a preview provider. It supports table DDL and partial safe model synchronization; migration runner, stored procedure synchronization, reset, and native lock support are not production-ready. The `modelsync` CLI therefore does not accept Oracle as a migration provider.
