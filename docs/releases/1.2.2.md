# ModelSync 1.2.2 - Integration Workflow Reliability and Release Gate Correction

Release date: 2026-07-01

ModelSync 1.2.2 is a release validation correction. It does not introduce a runtime breaking change. The release fixes the required integration workflow that blocked publication of the tagged but unpublished 1.2.1 build.

## Fixed

- MySQL CI service now creates the configured target database.
- MariaDB CI service now creates the configured target database.
- Stored procedure integration test is enabled in the required release workflow.
- Integration release gates now fail on unexpected skipped tests.
- Database images are pinned for reproducible CI.
- Target database readiness is validated before tests begin.

## Compatibility

- No ModelSync runtime breaking change.
- Existing 1.2.0 and 1.2.1-compatible source remains compatible.
- All six packages remain synchronized at version 1.2.2.

## Unpublished 1.2.1 tag

Version 1.2.1 was tagged but was not published to NuGet because the required
MySQL/MariaDB integration gate did not pass. Version 1.2.2 supersedes the
unpublished 1.2.1 tag.

## Validation Baseline

The release workflow pins database images for reproducible validation:

| Provider | Release validation image | Notes |
|---|---|---|
| MySQL | `mysql:8.4.10` | Validation baseline, not the full support range |
| MariaDB | `mariadb:11.8.8` | Validation baseline, not the full support range |
| PostgreSQL | `postgres:16` | Existing validation baseline |
| SQL Server | `mcr.microsoft.com/mssql/server:2022-latest` | Existing validation baseline |

The workflow verifies that the configured target database exists before running integration tests. It also parses TRX output and fails the release gate when an integration test is skipped unexpectedly.
