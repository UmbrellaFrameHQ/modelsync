# ModelSync 1.2.3 - SQL Server DBReset and Native Lock Fix

ModelSync 1.2.3 is a focused SQL Server reset reliability release.

## Fixed

- SQL Server `ResetDatabase=true` now runs the destructive database reset before acquiring the provider-native migration lock.
- Native migration lock release failures are logged without hiding the migration result.
- SQL Server reset can optionally create a backup before dropping the target database.

## Why It Matters

Applications that rebuild a database on startup can now run this sequence safely:

1. Reset target database.
2. Create schemas and history tables.
3. Synchronize table models.
4. Apply ordered SQL migration scripts.
5. Apply seed scripts.
6. Run again without reset and pass idempotently.

## New Reset Options

```csharp
var options = new MigrationRunnerOptions
{
    ResetDatabase = true,
    ResetOptions = new DatabaseResetOptions
    {
        Enabled = true,
        Approval = DestructiveOperationOptions.Allow(),
        ExpectedDatabaseName = "appdb",
        BackupBeforeReset = true,
        BackupDirectory = @"C:\SqlBackups",
        BackupFileName = "appdb-before-reset.bak"
    }
};
```

`BackupDirectory` and `BackupFilePath` are evaluated by SQL Server, so the path must be reachable by the SQL Server service account.

## Compatibility

No source breaking change is intended for 1.2.x consumers. Existing reset flows keep working with `BackupBeforeReset=false`.
