# ModelSync 1.3.0 - CLI, Dry-Run and Migration Reporting

ModelSync 1.3.0 is a feature release that makes migration execution easier to validate, preview, review, audit and communicate. It keeps the 1.2.x migration runner and provider API contracts intact while adding the first official CLI, dry-run workflows, reporting and clearer production guidance.

## Added

- `MigrationExecutionMarkdownReport` for human-readable deployment reports.
- `MigrationExecutionJsonReport` for CI systems, dashboards and structured release evidence.
- Provider support matrix with production and preview provider status.
- Migration reporting guide.
- CLI foundation with `modelsync version`, `modelsync validate`, `modelsync run --dry-run`, and report-writing `modelsync run`.
- DB-first scaffolder roadmap.
- Migration report, CLI quickstart, and GitHub Actions examples.
- Performance smoke coverage for large migration execution report rendering.

## Changed

- Oracle is explicitly documented as preview until the public NuGet package ID can be published with package-owner API-key permission.
- Documentation now positions ModelSync as a schema lifecycle toolkit rather than only a DDL generator.

## Compatibility

No source breaking change is intended for 1.2.x consumers. Existing migration runner, model synchronizer, stored procedure synchronizer, generator and analyzer APIs remain source-compatible.

## Why It Matters

Production migration tools need more than "success" or "failed". A deployment should be able to answer:

1. Which scripts were applied?
2. Which scripts were skipped?
3. Which script and batch failed?
4. Was a lock acquired?
5. Was history written?
6. How long did each item take?

1.3.0 adds that evidence and a safe CLI entry point without adding another migration engine or a DbUp dependency.
